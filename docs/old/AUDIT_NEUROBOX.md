# Аудит НейроБокс — карта реализации и план доработок

**Дата:** 2025-02-14  
**Версия бота:** 4.0  
**Путь к проекту:** `/opt/neurobox`

---

## 1. Резюме

Бот **работоспособен**: чат с AI, генерация изображений/видео/музыки/голос, документы, баланс, ЮKassa, админка, режимы чата, рефералы, FSM в Redis. Ниже — детальное сопоставление с целевым функционалом из промпта. Рабочий код не менялся; перечислены только расхождения и рекомендации.

## 13. Что не трогать (работает стабильно)

- Точка входа `bot/main.py`, роутеры, set_my_commands (start, restart, clear).
- Конфиг `config/settings.py`, загрузка из .env.
- БД: пул asyncpg, все текущие таблицы и индексы.
- ЮKassa: создание платежа, webhook payment.succeeded, проверка IP, идемпотентность по payment_id.
- Начисление/списание кредитов, рефералы при регистрации, ежедневный бонус (логика «в первый запрос дня»).
- Текст: провайдеры openai_text, chat_service (история в БД), режимы (mode_service, chat_modes).
- Изображения: маршрутизация по моделям, апскейл, регенерация, редактирование фото.
- Видео: text-to-video, image-to-video, выбор модели.
- Голос: STT, TTS, выбор голоса.
- Музыка: MusicGen.
- Документы: загрузка PDF/DOCX, 10 МБ.
- Баланс, покупка пакетов, Stars, CryptoBot, tx_history.
- Админка: вход по admin_ids, дашборд, статистика, рассылка (текст), пользователи, логи, система (базово).
- Middleware: BanCheck, RateLimit.
- Деплой: docker-compose (bot, worker, webhook, admin, postgres, redis).

## 14. План доработок (без поломки рабочего кода)

Приоритет — только добавление и точечные исправления по промпту.

### Высокий приоритет

1. **Webhook ЮKassa**  
   - Обработка `payment.canceled` (логирование в БД или admin_log).  
   - Обработка `refund.succeeded` (списание кредитов по refund amount / payment_id).

2. **Защита от фрода**  
   - Rate limit **5 платежей в минуту** на user_id (Redis счётчик + TTL 60 сек) при создании платежа в `balance.py`.

3. **Админ: неавторизованный доступ**  
   - На попытку /admin не из admin_ids: не отправлять ответ пользователю, только `log.warning` с user_id.

4. **Реферальная программа**  
   - Привести к «50 CR обоим»: REFEREE_CR=50, REFERRER_CR=50 (или оставить уровни, но базовые 50/50). Только конфиг/константы, без изменения схемы БД, если текущие поля позволяют.

### Средний приоритет

5. **История операций**  
   - В `tx_history` увеличить LIMIT с 15 до **20** и при необходимости добавить явный вывод типа/модели/кредитов в одну строку.

6. **Рассылка**  
   - Расширить FSM: опционально фото, опционально кнопка URL.  
   - Выбор аудитории: все / активные за 7 дней / платящие / с нулевым балансом.  
   - После рассылки: помечать пользователей с ошибкой «Forbidden» (заблокировали бота) в БД (например is_blocked=true), считать и показывать «заблокировано».  
   - Паузу между сообщениями привести к ~30 msg/сек (например 1/30 сек).

7. **Статистика админки**  
   - Добавить: ARPU, топ-5 моделей по использованию (по ai_requests), средний расход кредитов в день (по credit_transactions).

8. **Системный мониторинг**  
   - Проверка Redis (PING) и PostgreSQL (SELECT 1), вывод OK/FAIL.  
   - Последний успешный webhook YooKassa (последняя запись в payments с status=confirmed по времени confirmed_at).  
   - При наличии логов ошибок за 24ч — краткая сводка (можно позже, при появлении единого лога).

9. **Prompt injection**  
   - Перед вызовом текстовой модели: проверка промпта на подстроки вроде «ignore previous», «system:», «you are now» (регистронезависимо); при совпадении — отказ с нейтральным сообщением и логирование.  
   - Привести лимит длины промпта к 4000 символов в настройках (или оставить 10000, если продуктовая договорённость другая).

### Низкий приоритет

10. **Пакеты кредитов**  
    - Привести набор пакетов к 10 и ценам 29–8990₽ по промпту (только конфиг CREDIT_PACKS + миграции не требуются, если не меняем названия полей).

11. **Контекст чата в Redis**  
    - Опционально: хранить последние 15 сообщений в Redis с TTL 30 мин неактивности и читать оттуда (при желании полностью совпасть с промптом); иначе оставить текущую схему в БД.

12. **Настройки**  
    - Добавить в /settings выбор «длина ответа» (короткий/средний/длинный) и передавать в system_prompt или в параметры модели.

13. **Изображения**  
    - Кнопка «Убрать фон» под сгенерированной картинкой (вызов того же rmbg, что и в photo_tools).  
    - При необходимости заменить rembg на birefnet (fal.ai) по документации fal.

14. **Видео**  
    - Превью настроек и стоимости перед генерацией; при наличии API — выбор длительности/разрешения.

15. **Кросс-промо**  
    - Кнопка «Наши боты» в главном меню со списком 6 ботов.  
    - Контекстные подсказки (1 из 5 сообщений) по ключевым словам с ссылками на других ботов.

16. **Мелкое**  
    - metadata ЮKassa: добавить bot_name="neurobox" при создании платежа.  
    - Таблица payments: при следующих миграциях добавить currency, idempotence_key (опционально).  
    - Ежедневные 20 CR: при необходимости перенос на cron 00:00 MSK (отдельный воркер или скрипт).

## 15. Проверка после изменений

После любых правок:

1. Запуск: `docker compose up -d --build` (или только `restart` для bot/webhook при изменении кода).
2. Проверка /start, главного меню, баланса, выбора модели.
