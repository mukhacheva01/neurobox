После правок перезапустить бота: `docker compose restart neurobox_bot` (или `bot` в зависимости от имени сервиса).

## 3. Сильные стороны

- **Много моделей в одном месте**: текст (28 моделей), картинки, видео, голос, музыка — единый баланс CR и выбор модели.
- **Прозрачная тарификация**: стоимость в CR по каждой модели, бесплатные CR и купленные раздельно, пробный период (45 мин безлимит).
- **Режимы чата**: категории и пресеты системного промпта (режимы) + «Мой режим» (кастомный промпт).
- **Оценки ответов**: 👍/👎 с сохранением в `response_ratings` — база для аналитики качества.
- **Рефералы и промокоды**: механика есть в credits; уровни (Бронза/Серебро/Золото/Бриллиант) заданы в REFERRAL_LEVELS.
- **Кеш баланса**: Redis 90 с — снижение нагрузки на БД при частых проверках.
- **Документация**: IMPROVEMENTS.md, STATE.md, STRUCTURE.md, DEPLOY.md — удобно онбордить разработчика.

---

## 4. Слабые места и риски

1. **Структура документации в STRUCTURE.md устарела**: часть пунктов помечена «заглушка» (coming_soon), тогда как /search, /summary, /code, /ref, /promo, /feedback, /mode, /doc и др. уже реализованы в других модулях. Нужно синхронизировать STRUCTURE.md и гайд в боте с фактическими командами.
2. **Webhook ЮKassa**: в коде есть обработка вебхука оплаты, но бот на long polling. Для автоматического зачисления после оплаты нужен отдельный HTTP-endpoint для вебхука или явная документация, что подтверждение идёт вручную/по таймауту.
3. **Лимит длины промпта**: проверка есть (prompt_max_chars), но можно вынести в один хук и добавить подсказку «осталось N символов» там, где это возможно.
4. **Зависимость от имён моделей API**: при смене имён (как было с gpt-4.1-nano → gpt-5-nano) нужен маппинг; сейчас он есть в openai_text.py, но стоит периодически сверять с документацией провайдеров.
5. **FALAI_API_KEY в .env пустой**: картинки/видео не заработают без ключа fal.ai.

`cd /opt/neurobox && docker compose restart neurobox_bot`
