## 3) ПЛАН СТАБИЛИЗАЦИИ (SRE)

### Таймауты (ЗАДЕПЛОЕНО)
```
Компонент          Значение     Где
OpenAI/xAI/etc.    30s read     LLM_TIMEOUT = httpx.Timeout(connect=10, read=30, write=10, pool=10)
Anthropic          30s          httpx.Timeout(30.0) в AsyncAnthropic
PostgreSQL         default      asyncpg pool (5с connect по умолчанию)
Redis              3s           socket_timeout в rate_limit middleware
```

### Ретраи (БЫЛИ)
```
LLM:    max_retries=2, exponential backoff (2^attempt seconds)
        Ретраятся: 500, 503, RateLimitError
        НЕ ретраятся: 404 (NotFound), AuthenticationError
```

### Idempotency (ЗАДЕПЛОЕНО)
```
Платежи:   UNIQUE(payment_id) + confirm_payment_record() с FOR UPDATE + status check
Кредиты:   spend_credits() с FOR UPDATE в транзакции
Промокоды: FOR UPDATE на promocodes row
Callback:  owner_id проверка на всех кнопках (rate_up, clear_yes, tts_last)
```

### Circuit Breaker (ЗАДЕПЛОЕНО)
```
Тип:        Per-provider (openai/anthropic/google/deepseek/xai)
Threshold:  3 ошибки за 60 секунд
Cooldown:   60 секунд open state
Fallback:   openai→gemini-2.0-flash, anthropic→gpt-5-mini, google→gpt-5-nano
            deepseek→gpt-5-nano, xai→gpt-5-mini
Recovery:   Один успешный запрос → reset failures
```

### Graceful Degradation
```
Сценарий              Поведение
LLM provider down     Circuit breaker → suggest fallback model
PostgreSQL down       Бот крашится (P0 зависимость). Docker restart: always.
Redis down            Rate limit отключается (in-memory fallback). FSM → MemoryStorage.
ЮKassa down           "Платёжная система недоступна. Попробуйте позже." Бесплатные модели работают.
All LLM down          "Все модели временно недоступны." (circuit breaker на всех)
```

### Бэкапы (ЗАДЕПЛОЕНО)
```
Cron:   0 3 * * * /opt/neurobox/scripts/backup.sh
Format: pg_dump --format=custom --compress=6
Retain: 7 дней
Dir:    /opt/neurobox/backups/
```

### Релиз
```
Текущий процесс: правки в /opt/neurobox → docker compose restart
Код монтирован через volume → не нужна пересборка
Rollback: .bak файлы / git revert
Улучшение (TODO): canary через docker-compose profiles
```

## 4) НАБЛЮДАЕМОСТЬ

### SLI / SLO
```
SLI                       Метрика                          SLO (цель)
Доступность               % минут bot=Up                   99.5%
Latency текст p95         p95 elapsed в text handler        < 10 000 мс
Error rate LLM            ai_requests.status='error' / all  < 3%
Payment success rate      confirmed / total payments        > 95%
```

### Метрики LLM (в ai_requests table)
```
duration_ms       — latency per request
model             — distribution of model usage
status            — completed / error
credits_charged   — cost per request
provider          — openai/anthropic/google/deepseek/xai
```

### Метрики Telegram (в логах structlog)
```
"Unhandled error"          — global error handler
"reminder send failed"     — blocked users
"Rate limit"               — rate_limit middleware
```

### Метрики платежей
```
payments.status = 'pending'    — initiated
payments.status = 'confirmed'  — success
confirm_payment_record = None  — duplicate blocked
```

### Алерты
```
Уровень  Условие                               Канал
P0       Bot container down > 2 min             Docker healthcheck → restart: always
P0       Sentry: 50+ errors/hour                Sentry alert → email
P1       payment confirmed = 0 за 24ч           Daily report → TG admin
P2       DAU drop > 30% vs prev week            Daily report → TG admin
```
