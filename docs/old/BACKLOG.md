# Backlog на 1–3 дня

Приоритеты: **P0** (критично), **P1** (важно), **P2** (желательно).

---

## Стабильность

| ID | Задача | Приоритет | Критерий готовности |
|----|--------|------------|---------------------|
| S1 | Мониторинг ошибок в проде (Sentry уже подключён — проверить алерты) | P1 | Критичные ошибки приходят в Sentry, алерт при всплеске |
| S2 | Healthcheck endpoint или скрипт для оркестратора (есть `scripts/healthcheck.py`) | P2 | K8s/cron проверяют живой бот и БД |
| S3 | Обратимая миграция БД: проверять все новые миграции на rollback | P1 | У каждой миграции есть обратный шаг или документированный rollback |

## Чек-лист ручного smoke (как юзер)

Перед релизом или после деплоя прогнать в Telegram:

1. **/start** → появилось меню с кнопками.
2. Нажать 2–3 кнопки (например «Текст», «Картинки») → экраны открываются.
3. **Назад / Отмена** → возврат в главное меню.
4. Отправить **неизвестную команду** (например `/xyz`) → ответ без падения.
5. Отправить **пустое сообщение** (или только пробелы) → ответ без падения.
6. Отправить **очень длинное сообщение** (около лимита) → либо принято, либо понятная ошибка.
7. **5 быстрых кликов** по одной callback-кнопке → нет дублей и ошибок.
