# НейроБокс — Тесты, сценарии и критерии релиза

**Вводные (авто-детект):** BOT_NAME: НейроБокс. GOAL: 40+ нейросетей в одном боте (текст, картинки, видео, голос, музыка). ЦА: пользователи Telegram, RU/EN. Язык: RU. Платформа: Telegram Bot API, UI: команды + inline + reply-меню. Команды: /start, /restart, /clear, /admin, /model, /balance, /video, /mode, /setprompt, /privacy, /terms. Интеграции: ЮKassa, CryptoBot, PostgreSQL, Redis, OpenAI/Anthropic/Google/DeepSeek/xAI, fal.ai, Sentry. Монетизация: кредиты (CR), 20 CR/день, триал 45 мин, пакеты. Лимиты: 45/60 сек rate limit, текст 4000 символов.

---

## 1) КАРТА ПРОДУКТА

**User-flows:** /start → регистрация → меню → Чат с AI / Картинка / Видео / … → списание CR → ответ; Пополнить → пакет → оплата → webhook → зачисление; Триал 45 мин → безлимит → истечение → снова баланс; ◀️ Меню и /cancel из любого экрана.

**Сущности:** Пользователь (users), кредиты/баланс, подписка/триал, платёж (payments), история чата, события (events).

**Состояния юзера:** новый, активный (есть CR/триал), безлимит, нет баланса (paywall), бан (is_blocked).

---

## 2) ТЕСТ-МАТРИЦА

- **Functional:** старт, меню, чат, картинки, видео, голос, музыка, документы, инструменты, режимы, баланс, модели, рефералы, гайд, админка.
- **UX:** тон, ошибки без тех. деталей, подсказки (неизвестная команда, пустое, длинное), /cancel, устаревшие callback.
- **Payments:** отображение цен, переход на оплату, webhook идемпотентность, возвраты при ошибке, подтверждение зачисления.
- **Security:** rate limit 45/60 сек, prompt injection фильтр, gen_lock и owner_id для callback.
- **Reliability:** таймауты LLM, retry 2 при 5xx/RateLimit, circuit breaker и fallback.
- **Performance:** латентность, 429 throttle.
- **Data integrity:** дубли callback (lock), повтор webhook (payment_lock).
- **Admin:** доступ по ADMIN_IDS, дашборд, рассылка, блокировка.
- **Analytics:** события в events, track() не падает.

## 4) TEST CHAT SCRIPTS

**Скрипт A (онбординг + успех):**
- USER: /start → BOT: приветствие, баланс, меню. PASS: есть кнопки Чат с AI, Картинка, Баланс.
- USER: нажать «💬 Чат с AI» → BOT: «Напиши вопрос» или аналог.
- USER: Привет → BOT: ответ LLM, внизу −CR, остаток. PASS: один ответ, списание 1+ CR.
- USER: ◀️ Меню → BOT: главное меню.
- USER: «🎨 Картинка» → BOT: запрос промпта.
- USER: Кот в космосе → BOT: картинка или понятная ошибка.
- USER: Меню → Баланс → BOT: баланс, Пополнить.
- USER: /clear → BOT: подтверждение «Очистить историю?»
- USER: Да → BOT: «очищено» или аналог.
- USER: /start → BOT: меню (повторный старт).

**Скрипт B (ошибки):**
- USER: /qwerty → BOT: «Неизвестная команда. Нажми /start для меню.»
- USER: (пустое) → BOT: «Напиши текст сообщения или нажми /start для меню.»
- USER: (строка 4001 символ) → BOT: «Слишком длинное сообщение (макс. 4000 символов). Сократи.»
- USER: Чат → только пробелы → BOT: подсказка ввести текст.
- USER: в выборе модели /cancel → BOT: «❌ Отменено.» и меню.
- USER: нажать старую inline-кнопку → BOT: «Сообщение устарело. Нажми /start.»

**Скрипт C (лимиты/оплата):**
- Пополнить → список пакетов с ценами.
- Выбор пакета → открывается ссылка оплаты (реально не платить).
- Кнопка «🚀 45 мин безлимита» → подтверждение → безлимит активен, в меню таймер.
- Несколько запросов в чат в триале → ответы без списания CR.
- После триала /start → снова баланс CR.
- При 0 CR платное действие → paywall «Пополнить».
- [Sandbox] Оплата тестом → возврат в бота → «Кредиты зачислены», баланс вырос.

**Скрипт D (деградация):**
- Чат при отключённом LLM или моке 500 → BOT: «Ошибка API» / «Попробуйте позже», при refund — «Кредиты возвращены».
- 10 сообщений подряд быстро → после 45/мин: «Слишком много запросов. Подожди около минуты.»
- Дважды отправить один и тот же webhook payment.succeeded (тест) → зачисление один раз.

## 5) EDGE CASES

Повторный /start; callback storm (5–10 кликов); один update дважды; сообщение >4000; вложение вместо текста; emoji-only; ссылки/@; смена языка/таймзоны; пользователь без username; блокировка бота (403 при рассылке, is_blocked).

---

## 6) RELEASE CRITERIA

P0 кейсы пройдены; pytest зелёный; healthcheck OK; нет массовых ошибок в логах; деплой без падения бота; smoke A и B вручную.

## 7) OBSERVABILITY

Метрики: запросы/ошибки по провайдерам. Алерты: Sentry. Логи: structlog, request_id/user_id, без секретов. Проверки: healthcheck, tail логов после деплоя.

## 8) TOP 10 LIKELY BUGS

1. Двойное списание при двойном callback — lock + owner_id. 2. NameError при старте (неимпортированный F) — тест main import. 3. daily_stats схема — (date, metric, value). 4. Двойное зачисление webhook — payment_lock. 5. Пустое/неизвестная команда без ответа — обработчики. 6. Refund при ошибке LLM — _refund_if_needed везде. 7. Обход rate limit — единый счётчик. 8. Analytics log event clash — event_name=. 9. Устаревший callback без answer — stale_router. 10. Таймаут без ответа юзеру — wait_for + сообщение + refund.
