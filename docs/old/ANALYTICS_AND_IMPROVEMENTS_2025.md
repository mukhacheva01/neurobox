# НейроБокс — аналитика и предложения по улучшению

**Дата:** февраль 2025  
**Версия бота:** 4.0

---

## 1. Текущее состояние

### Архитектура

| Компонент | Технология |
|-----------|------------|
| Бот | aiogram 3.x, long polling, FSM (Redis или Memory) |
| БД | PostgreSQL (asyncpg), миграции в `migrations/` |
| Кеш | Redis (FSM, кеш баланса 90 с) |
| Текст | OpenAI, Gemini, Claude, DeepSeek, xAI — без маппинга, провайдер по префиксу `model_id` |
| Медиа | fal.ai (картинки, видео), MusicGen/Suno, Edge TTS, OpenAI TTS, Whisper |
| Оплата | ЮKassa, webhook на порту 8090 |
| Фон | arq worker, напоминания в основном процессе |

### Реализованный функционал

- **Чат:** 28+ текстовых моделей, стриминг для GPT/Grok, контекст (до 20 сообщений), режимы и «Мой режим», лимит промпта (10k символов), оценки 👍👎.
- **Медиа:** картинки, видео (Kling, Veo, Sora и др.), голос (TTS/STT), музыка, фото-инструменты (rmbg, ocr; /style — заглушка).
- **Деньги:** кредиты (бесплатные 20/день + купленные), тарифы по моделям, пробный период 45 мин, покупка паков, подписки с напоминанием об окончании.
- **Социальное:** рефералы (уровни Бронза/Серебро/Золото/Бриллиант заданы, множитель в коде есть), промокоды, /feedback в БД.
- **Админка:** дашборд, статистика (пользователи, генерации, выручка), экспорт CSV, отчёт по оценкам 👍👎 за 7/30 дней, пользователи, финансы, промокоды, рассылка, модерация.

### Сильные стороны

- Один бот — много провайдеров текста и медиа, единый баланс CR.
- Прозрачная тарификация (CR по модели), разделение бесплатных и купленных кредитов.
- Кеш баланса в Redis, возврат CR при ошибке генерации.
- Документация: STRUCTURE.md, DEPLOY.md, гайд в боте.
- Webhook ЮKassa вынесен в отдельный сервис — оплаты зачисляются автоматически.

## 2. Слабые места и риски

### Продукт и UX

| Проблема | Где | Влияние |
|----------|-----|--------|
| Дефолтная модель в БД `gpt-4.1-nano` | init.sql, users.text_model | У новых пользователей может не совпадать с актуальным списком (сейчас в конфиге `gpt-5-nano`). |
| Нет явного онбординга | start.py | Новый пользователь видит меню, но не получает подсказку «напиши сообщение — получи ответ» или «выбери модель в /model». |
| /style для фото — заглушка | photo_tools.py | В гайде упоминается, но не работает. |
| Длинные ответы обрезаются | text.py, response_max_chars=4000 | Часть ответа может теряться без пояснения «продолжение обрезано». |
| Нет retry при временных ошибках API | openai_text.py, falai_* | Один 5xx/rate limit — пользователь сразу видит ошибку, кредиты возвращаются; повторной попытки нет. |

### Технические

| Проблема | Где | Влияние |
|----------|-----|--------|
| Rate limit в памяти | rate_limit.py | При перезапуске счётчики сбрасываются; при нескольких инстансах бота лимиты не общие (для одного инстанса — ок). |
| FSM в Memory при недоступности Redis | main.py | Состояния теряются при рестарте; для продакшена лучше требовать Redis. |
| Ключи в .env без валидации при старте | config/settings.py | Бот стартует и падает уже в хендлере при отсутствии ключа (например FALAI для картинок). |
| Нет метрик/health-check | — | Сложнее мониторить доступность и нагрузку. |

### Бизнес и контент

| Проблема | Где | Влияние |
|----------|-----|--------|
| Реферальный множитель по уровню не везде применён | credits.py | get_referral_level есть; при начислении бонуса рефереру нужно явно умножать на множитель. |
| Нет A/B или фич-флагов | — | Нельзя постепенно раскатывать изменения по части пользователей. |

## 3. Предложения по улучшению

### Высокий приоритет (быстрые победы)

| № | Предложение | Где менять | Действие |
|---|-------------|------------|----------|
| 1 | **Дефолтная текстовая модель** | migrations: новая миграция или init | Для новых пользователей выставлять `text_model = 'gpt-5-nano'` (или актуальную бесплатную из config). Для существующих — опциональная миграция UPDATE. |
| 2 | **Краткий онбординг после /start** | start.py | После приветствия отправлять второе сообщение: «Просто напиши вопрос в чат. Модель можно сменить: /model» (или одну кнопку «Выбрать модель»). |
| 3 | **Подсказка при обрезке ответа** | text.py | Если ответ обрезан по response_max_chars — в конец добавлять «\n\n… (ответ обрезан, продолжай в следующем сообщении)» или кнопку «Продолжить». |
| 4 | **Проверка ключей при старте** | scripts/entrypoint.sh или main.py | Опционально: при SERVICE_TYPE=bot вызывать лёгкую проверку (например list models или один короткий запрос) для OpenAI/Anthropic/Google/DeepSeek/Grok; при отсутствии ключа — warning в лог, не падать. Либо вынести в `scripts/check_apis.py` и вызывать вручную после деплоя. |

### Средний приоритет (функционал)

| № | Предложение | Где менять | Действие |
|---|-------------|------------|----------|
| 5 | **Реализовать /style для фото** | photo_tools.py, falai_photo / falai_image | Либо img2img по описанию стиля, либо фиксированные стили кнопками под фото; списание CR. |
| 6 | **Реферальный уровень в UI** | ref_promo_stats.py | Множитель уже применяется в credits.py при начислении. В /ref показывать текущий уровень (Бронза/Серебро/…) и прогресс до следующего (например «ещё 2 друга до Серебра»). |
| 7 | **Retry при временных ошибках API** | openai_text.py, при необходимости falai_* | 1–2 повтора с backoff при RateLimitError / 503; не возвращать CR сразу при первой ошибке. |
| 8 | **Документы: приём PDF/DOCX** | doc_handler.py | Хендлер на F.document → извлечение текста (doc_extract уже есть) → суммаризация/вопросы в LLM; лимит размера 5–10 MB. |
| 9 | **Кнопка «Апскейл» под картинкой** | image.py | Inline-кнопка под сгенерированным изображением → upscale через fal.ai, списание CR. |

### Низкий приоритет (качество и масштаб)

| № | Предложение | Где менять | Действие |
|---|-------------|------------|----------|
| 10 | **Rate limit в Redis** | rate_limit.py | Хранить счётчики в Redis по ключу `ratelimit:{user_id}` со скользящим окном; при нескольких инстансах лимиты будут общими. |
| 11 | **Health-check endpoint** | webhook_server или отдельный сервис | GET /health → проверка подключения к БД и Redis; для балансировщика или мониторинга. |
| 12 | **Счётчик символов промпта** | text.py или middleware | При длине промпта > 7–8k показывать в ответе на «слишком длинный»: «Сейчас N символов, нужно не более M». |
| 13 | **Логирование стоимости в CR по моделям** | credits.py, ai_requests | Уже есть ai_requests; при желании — дашборд админки «топ моделей по расходу CR» за период. |
